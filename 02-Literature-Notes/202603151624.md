#### LIBUV

V8 e LIBUV vao alternando como trabalham, nao trabalham em paralelo

Event Loop = So trabalha quando a call stack esta fazia 

Promises tem 3 estados: Pending, Fulfilled, Reject.
Promises nao sao lazys, executam quando sao chamadas
Event loop orquestra para que uma callback seja executada assim que a promise for resolvida, se o V8 estiver ocupado a promisse mesmo estando completa ele nao vai rodar. Ele vai para a microstask Queue -> so sai da fila e vai para a callstack quando o v8 parou trabalhar.
